# Design Learnings — MAISON IVOIRE

「純白 `#ffffff` を CSS に一切書かない」という縛りは、想像以上にレイアウト判断を強制した。
ページ地を Ivoire `#f4f0e8`、セクション帯を Marbre `#e8e4dc`、引用と予約カードを Vélin `#ede8de` に
分配した瞬間、セクション境界に罫線を引く必要が消え、**背景の階調差そのものが構造になる**。
hairline `1px solid #c9c3b6` は装飾ではなく「同じ階調内での補助線」として再定義された。

ローズ `#c9a999` を「リンク下線・JOURNAL の日付・引用記号」だけに閉じ込めると、
ページ全体での体温が一気に上がる。**色は面積ではなく "登場の希少さ" で温度を持つ**。
ボタン塗りや見出しに使った瞬間に romantic に転ぶため、Reservation ボタンは白地＋鋼の枠線のみで成立させた。

スタッフ表記を「姓のみ＋手と後ろ姿マクロ」に統一すると、自己紹介文の文体が自動的に決まった。
「○○年から在籍」「○○の修練を経て」という淡々とした事実列挙だけが許される。
**顔写真を捨てることで、コピーから誇張表現が消える**という構造的な効用がある。

メニュー表現では、価格・所要時間・名称・説明の四列を grid で並べ、罫線は hairline のみに統一。
モバイルでは grid-template-areas で「番号 / 名前 / 説明 / 時間と価格」に再構成し、
価格を右寄せに置くことで明朝の整列の品位を保てた。Cormorant の tabular-nums は数値整列の主役だった。

Cormorant Garamond 300 は美しいが、モバイル `<768px` で確実に線痩せする。
`@media` で h1 と blockquote を 400 に昇格させると視認性が戻る。
**ウェイトは固定値ではなく、ビューポートに応じた可変設定として扱うべき**という気づきが残った。

最大の学びは、**「白の階調を主役にする」と決めると、写真選びの基準が一意になる**こと。
Unsplash で「marble / silver scissors / north window / hair macro」を撮影者の彩度感で選別する作業は、
配色を考える作業と完全に一致した。spec の禁止事項が、写真選定の判断基準そのものになっていた。

— *Vol.03 / 2026, MAISON IVOIRE build notes*
